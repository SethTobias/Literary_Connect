Static:
HTTP Methods Path Description
GET / (Home/Static Page) Path to Static Page detailing what each path is responsible for handling in the routes.
Users:
HTTP Methods Path Description
GET /user Path is responsible for retrieving all users data within the API.
GET /user/:user_id Path is responsible for retrieving a users data from the API specifically by their user_id, specified in the path.
POST /user/register Path is responsible for retrieving adding a new users data into the API.
POST /user/login Path is responsible for retrieving the users data from the API. It also assigns and verifies a token to the specific user.
PATCH /user/edit/:user_id Path is responsible for allowing the user to make any changes to their data within the API specifically their own personal data.
PATCH /user/editType/:user_id Path is responsible for allowing the user to make changes to their account type this however is not set permanently as an admin must first approve of this change.
PATCH user/admin/edit/:user_id Path is responsible for allowing an admin to make account changes to any user accounts.
PATCH /user/report Path is responsible for altering the user account status to Reported. Which flags said user for any inappropriate behaviour.
DELETE /user/delete Path is responsible for allowing data about the user to be deleted from the API.
Users Follows:
HTTP Methods Path Description
GET /user/follows Path is responsible for retrieving all follows data within the API.
GET /user/following/:user_id Path is responsible for retrieving all the users followed by a specific user.
GET /user/followers/:user_id Path is responsible for retrieving all the users that are following a specific user.
POST /user/follow Path is responsible for allowing new follower/following data to be added into the API.
DELETE /user/unfollow Path is responsible for allowing data about the follower/following to be deleted from the API.
Post:
HTTP Methods Path Description
GET /post Path is responsible for retrieving all posts data within the API.
GET /post/:post_id Path is responsible for retrieving a posts data within the API by a specific post_id.
GET /post/:user_id Path is responsible for retrieving all posts of a specific user within the API.
POST /post/add/:user_id Path is responsible for allowing new post data to be added into the API by a specific user.
PUT/PATCH /post/edit/:post_id Path is responsible for allowing data about a specific post to be altered within the API.
DELETE /post/delete/:post_id Path is responsible for allowing data about a specific post to be deleted from the API.
Post Likes:
HTTP Methods Path Description
GET /post/like/:post_id Path is responsible for retrieving all likes of a specific post within the API.
POST /post/like/add/:post_id/:user_id Path is responsible for allowing a new like to be added into the API assigning it to a specific post and specific user.
DELETE /post/like/delete/:like_id Path is responsible for allowing like data to be deleted from the API.
Comment:
HTTP Methods Path Description
GET post/comment/:post_id Path is responsible for retrieving all comments data about a specific post within the API.
POST post/comment/add/:post_id/:user_id' Path is responsible for allowing new comment data to be added into the API.
PUT/PATCH post/comment/edit/:comment_id Path is responsible for allowing data about the comment to be altered within the API.
DELETE post/comment/delete/:comment_id Path is responsible for allowing data about the comment to be deleted from the API.
